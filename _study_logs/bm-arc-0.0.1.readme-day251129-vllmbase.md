## vllm 的特点

- vllm 意义在于通过 PagedAttention 和连续批处理等技术，即使是在满负荷高并发的情况下，尽可能高的提升每一个请求的处理速度以提升吞吐量；

  关于模型吞吐量的解释 https://deepwiki.com/search/-3090-13b_d55750b5-6567-4290-a5f5-08080167b1fb

- PagedAttention 将 KV 缓存分割成固定大小的块，避免了传统方法中的内存碎片和预分配浪费，让系统能够：动态分配内存给活跃请求；减少内存浪费，支持更多并发请求；提高内存访问效率

- 连续批处理的智能调度：连续批处理技术让 vLLM 能够动态处理不同长度的请求；优先处理解码请求：确保正在生成的请求能快速完成；智能批处理：将计算密集型（预填充）和内存密集型（解码）操作合理搭配；动态调整：根据可用资源动态调整批次大小

- 底层 CUDA 优化，这些内核专门为 PagedAttention 设计，能够：高效处理分块的 KV 缓存；支持不同的数据类型和量化格式；针对不同硬件平台优化
